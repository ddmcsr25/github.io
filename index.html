Android FRP Bypass Vulnerability on Pixel 6

Author: Derek Morris
Reported to: Google Android Security Rewards Program
Date Reported: June 2024
Status: Won’t Fix (Intended Behavior)
Reward: Issued
Build Fingerprint: google/oriole/oriole:14/AP2A.240605.024/11860263:user/release-keys

Overview

This report documents a critical security flaw on a Google Pixel 6 running Android 14. The flaw allows a user to bypass Factory Reset Protection (FRP) — even with a locked bootloader and USB debugging disabled — by sideloading a beta OTA update via recovery mode. The vulnerability was responsibly disclosed and rewarded by Google, but marked as “Won’t Fix (Intended Behavior).”

Exploit Impact

With physical access to a locked device, an attacker can:

  • Bypass FRP and user PIN security
  • Re-enroll the device under a new Google account
  • Access developer options and unlock the bootloader
  • Install custom or factory ROMs
  • Use or resell the device without authorization

Reproduction Steps

Tested on Pixel 6 (Android 14, June 2024 security patch). Required tools:

  • Android Platform Tools (ADB)
  • Pixel Flasher Tool
  • Android 15 Beta 3.1 OTA image
Full steps:
  1. Boot into recovery and factory reset the device.
  2. Complete setup with a Google account and PIN; confirm FRP is active.
  3. Reboot into recovery mode → Select “Apply update from ADB.”
  4. Connect to PC via USB → Use Pixel Flasher to sideload Android 15 beta image.
  5. Once flashed, reboot the device → Setup no longer enforces previous account or lock.
  6. Developer options and OEM unlocking now accessible.

Root Cause

Android’s ADB sideload implementation does not properly enforce FRP state post-flash. The process resets the security state without re-verifying the account or PIN lock after flashing an OTA image.

Google's Response

The vulnerability was reviewed and rewarded under the Android Security Rewards Program. However, it was marked as “Won’t Fix (Intended Behavior).”

“Thank you for your submission. This behavior is considered intended and will not be addressed. However, we are issuing a reward under the Android Security Rewards program.”
